# 消息推送
作为一个IM应用，需要拥有**消息推送**的能力。就你收到消息后，弹通知给你，让你知道你有新消息。

## 技术选择
根据需求，很容易想到使用**WebSocket**或者**TCP长连接**来接受服务器的消息推送，再不济使用HTTP轮询，获取到新消息后弹通知给用户。

这样的的方式在桌面平台是可行的，只要应用在后台存活，就可以保持长连接来实现消息推送。但是在移动平台，由于电池优化策略，应用无法在后台长时间保活，长连接也无法保持。所以就需要一些特别的手段来实现消息推送。

### 原生Android
Android有一套Google实现的推送系统**FCM**（*Firebase Cloud Messaging*），其基于**GMS**（*Google Mobile Services*）实现，GMS是系统级服务，可以保持长连接，应用便可以使用其来推送消息。

### 国产Android
但是，由于国内Google服务访问受限，并且国产ROM剔除了大部分Google服务，导致FCM不能在国内使用。各家厂商的ROM实现了自己的推送服务，可以使用统一推送联盟封装的SDK完成消息推送。

### iOS
Apple也有一套推送系统**APNS**（*Apple Push Notification Service*），其基于系统级的长连接，并且Apple在国内业务不受限，可以方便地使用APNS来推送消息。
